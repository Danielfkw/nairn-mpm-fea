Deprecated Commands

The following commands have been deprecated (in alphabetical order). The most common reasons are to change the default units for the command or to change the style of the input. These commands all still work.

<ArchiveTime maxProps="(props)">(time)</ArchiveTime>
Replaced by <ArchiveGroup>. Enter (time) for the time interval between archiving (in time units or in units set by a optional units attribute). The (props) attribute forces archiving if (props) propagations and/or debonds have occurred since the last archive (default is 0, which disables counting of propagations). Deprecated in 2.0.0.
<CrackList friction='0.1' Dn='200' Dnc='-1' Dnt='200' Dt='5'> ... </CrackList>
This command used to set custom crack contact properties (by methods explain below). All the above attributes have been replaced by law and lawname attributes to set properties using a contact law.
<FirstArchiveTime>(first time)</FirstArchiveTime>
Replaced by <ArchiveGroup>. Enter (first time) for the first time results are archived (in time units or in units set by a optional units attribute). Deprecated in 2.0.0.
<Friction Dt='(Dt)' Dn='(Dnt)' Dnc='(Dnc)' mat='(num)' matname='(name)'>(frict)</Friction>
This command sets friction or imperfect interface properties and may be used in the <MultiMaterialMode> block, in the <Crack> block, or in a material definition block. All the above attributes have been replaced by law and lawname attributes to set properties using a contact law. The former command settings are:
  • (frict)=(zero or positive number) - contact is by friction with the supplied coefficient of friction (which defaults to 0).
  • (frict)=(or a number between -1 and -9) - contact by stick conditions.
  • (frict)=(or a number less than -10) - reverts to a single velocity field. A better approach to contact with a single velocity field is to have two or more materials share the same velocity field.
  • (frict)=11 - converts multimaterial interactions to be an imperfect interface with interface parameters supplied in (Dt), (Dnt), and (Dnc) for tangential direction and normal direction in tension or compression, respectively. If (Dnc) is not give it is set equal to (Dnt) (interface parameters in pressure/length units).
  • (mat) (or (matname)) - sets custom contact properties (from settings in other parameters) to use for contact between current material and a second material, which is specified using number or name. These attributes are only allowed when the command is in a material definition block.
In a <MultiMaterialMode> block, the parameters set default multimaterial mode contact properties. In a <Crack> block, the parameters set default multimaterial mode contact properties. In a material definition block, the commands set custom contact properties between two materials.
<MaxTime>(max time)/>
Replaced by <Timing>. Enter (max time) for the time that the calculations will stop (in time units or in units set by a optional units attribute). Deprecated in 2.0.0.
<TimeFactor>(time factor)/>
Replaced by <Timing>. Enter (time factor) for the Courant-Friedrichs-Levy (or CFL) factor to use when determining the MPM time step. The time step used in the analysis will be the minimum of the one determined using this factor and the one entered in (time step). The default CFL is 0.5. Deprecated in 2.0.0.
<TimeStep>(time step)/>
Replaced by <Timing>. Enter (time step) for the MPM time step (in time units or in units set by a optional units attribute). Deprecated in 2.0.0.
